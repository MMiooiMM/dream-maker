# AGENTS.md

## 目的
確保每次對程式碼或文件的變更，都會同步反映在 `README.md`，讓專案說明與實際狀態一致。

## 適用範圍
本檔案位於專案根目錄，規範適用於整個 repository。

## 變更與 README 同步規則
當你修改任何檔案時（程式碼、設定、文件、腳本、資料），在提交前必須完成以下檢查：

1. **判斷是否影響 README 內容**
   - 任何會影響使用者理解或操作的變更，都視為需要更新 README。
   - 常見例子：
     - 新增 / 移除功能
     - 指令、參數、路徑、檔名、目錄結構改變
     - 安裝步驟、環境需求、相依套件改變
     - 輸出格式、工作流程、UI 操作流程改變
     - 文件結構重整（例如說明頁面移動位置）

2. **同步更新 `README.md`**
   - 更新對應章節（例如「目錄結構」「工作流程」「環境需求」「技術棧」「詳細說明」）。
   - 僅寫入與實作一致、可驗證的資訊；避免陳舊或推測內容。

3. **在提交訊息中明確標示 README 狀態**
   - 若有更新 README，請在 commit message 或 PR 描述中加入：`docs: README updated`。
   - 若判斷不需更新 README，請在 commit message 或 PR 描述中加入：`docs: README no change needed`，並附一句理由。

## 強制檢查清單（提交前）
- [ ] 我已檢查本次變更是否影響 README。
- [ ] 若有影響，我已更新 `README.md`。
- [ ] README 中提到的路徑、指令、功能名稱與現況一致。
- [ ] 我已在 commit message 或 PR 描述標示 README 更新狀態。

## 優先原則
若任務指示與本文件衝突，以更高優先層級的指示（system / developer / user）為準；其餘情況一律遵守本規範。
